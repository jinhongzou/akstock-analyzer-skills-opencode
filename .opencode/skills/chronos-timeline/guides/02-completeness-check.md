# 第一步：事件完整性检查

## 分析目标

检查时间线采集到的事件是否能完整覆盖该事件的完整生命周期。从**覆盖度**、**连续性**、**关键节点**三个维度评分。

## 评分标准（0-100 分）

### 一、覆盖度（40 分）

评估事件是否采集到了该事件主线上的各个阶段事件。

| 分数 | 标准 |
|------|------|
| 35-40 | 所有预期阶段都有对应事件，无遗漏 |
| 25-34 | 大部分阶段覆盖，缺少 1-2 个非关键阶段 |
| 15-24 | 缺少核心阶段，只能拼凑出大致脉络 |
| 0-14 | 事件严重缺失，无法形成完整链条 |

### 二、连续性（30 分）

评估事件之间的时间间隔是否合理，有没有明显断裂。

| 分数 | 标准 |
|------|------|
| 25-30 | 事件之间连续性好，没有超过 1 年的空白期 |
| 15-24 | 有少量跳跃（6-12 个月空白），但不影响理解主线 |
| 5-14 | 有明显的长时间空白期（超 1 年），主线只能凭推测 |
| 0-4 | 事件极度零散，几乎无法建立连续叙事 |

### 三、关键节点覆盖（30 分）

检查事件首次出现、重要转折、最终结果等关键节点是否包含在内。

| 分数 | 标准 |
|------|------|
| 25-30 | 所有关键节点（启动/转折/结果）都有事件覆盖 |
| 15-24 | 包含大部分关键节点，但缺少 1-2 个 |
| 5-14 | 缺少多个关键节点，只能拼凑出大致情况 |
| 0-4 | 关键节点大面积缺失，无有效分析基础 |

## 计分方法

**总分 = 覆盖度得分 + 连续性得分 + 关键节点得分**

## 扣分项

| 情况 | 扣分 |
|------|------|
| 关键词过窄导致大量相关事件未匹配 | -10 |
| 事件分布极度不均（90% 集中在某一年） | -5 |
| 日均事件数超过 2 条（噪声过多的信号） | -5 |
| 新闻事件占 60% 以上（缺少官方公告） | -5 |

## 输出格式

```
## 第一步：事件完整性检查

### 评分明细
- 覆盖度：32/40 — 理由...
- 连续性：25/30 — 理由...
- 关键节点：20/30 — 理由...

### 总分：77/100

### 结论
事件完整性 [优秀/良好/一般/不足]

### 缺失信号（如有）
- 建议补充搜索的方向...
```
